# SmartScore v2 iPad 테스트 체크리스트

## 1. 사전 준비

### 1.1 Python 복원 서버 실행 (PC/Mac)

1. 터미널에서 프로젝트 루트로 이동:
   ```bash
   cd modules/C_score_image_restoration
   ```

2. 가상 환경 활성화 (선택사항이나 권장):
   ```bash
   python -m venv venv
   source venv/bin/activate   # macOS/Linux
   venv\Scripts\activate      # Windows
   ```

3. 의존성 설치:
   ```bash
   pip install -r requirements.txt
   ```

4. 서버 실행 (모든 네트워크 인터페이스에서 접근 허용):
   ```bash
   python server.py --host 0.0.0.0 --port 8888
   ```

5. 서버 정상 동작 확인:
   ```bash
   curl http://localhost:8888/api/health
   ```
   `{"status": "ok"}` 응답이 오면 정상

### 1.2 Flutter Web 빌드 및 실행

1. 프로젝트 루트의 `build_unified` 디렉토리로 이동:
   ```bash
   cd build_unified
   ```

2. 의존성 설치:
   ```bash
   flutter pub get
   ```

3. Web 빌드 및 실행 (호스트 바인딩 포함):
   ```bash
   flutter run -d chrome --web-hostname 0.0.0.0 --web-port 8080
   ```
   또는 릴리스 빌드 후 서빙:
   ```bash
   flutter build web
   python -m http.server 8080 -d build/web
   ```

### 1.3 iPad에서 접속

1. iPad와 PC/Mac이 **같은 WiFi 네트워크**에 연결되어 있는지 확인
2. PC/Mac의 IP 주소 확인:
   - macOS: `ifconfig | grep "inet " | grep -v 127.0.0.1`
   - Windows: `ipconfig` 에서 IPv4 주소 확인
   - Linux: `hostname -I`
3. iPad Safari에서 접속: `http://<PC의_IP주소>:8080`
4. 앱의 설정 화면에서 복원 서버 주소를 `http://<PC의_IP주소>:8888`로 변경

### 1.4 테스트 이미지 준비

- 일반 악보 사진 (3-5MB): 기본 테스트용
- 고해상도 악보 사진 (15-25MB): 대형 이미지 경고 테스트용
- 저해상도 악보 사진 (<200x200px): 최소 크기 오류 테스트용
- 기울어진 악보 사진: 기울기 보정 테스트용
- 그림자가 있는 악보 사진: 그림자 제거 테스트용
- PDF 파일이 아닌 이미지 (JPG, PNG): 형식 호환성 테스트용
